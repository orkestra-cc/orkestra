# `docs/site/` — canonical source for docs.orkestra.cc

This directory is the **source of truth** for the hand-written pages of [docs.orkestra.cc](https://docs.orkestra.cc).

The site itself is a Docusaurus 3 project in a separate repository ([`orkestra-cc/orkestra-docs`](https://github.com/orkestra-cc/orkestra-docs)) — but its content is no longer hand-edited there. Instead, the docs build runs a sync script (`scripts/sync-monorepo-docs.ts` in `orkestra-docs`) that mirrors this directory into the site's `docs/` tree at every build.

## Why monorepo as source

- A visitor browsing this repo on github.com sees the same operator guides this site renders — no context switch to a separate docs project.
- Docs edits flow through the same PR queue as code, with the same reviewers, same `make ci` gates, and same CODEOWNERS.
- The docs renderer can be swapped (Docusaurus → mdBook → anything else) without touching content.

## What lives here

| Path | Becomes |
|---|---|
| `intro.mdx` | Site home (`/`) |
| `getting-started/*` | `/getting-started/*` |
| `architecture/*` | `/architecture/*` |
| `operating/*` | `/operating/*` |
| `sdk/*` | `/sdk/*` |
| `contributing/*` | `/contributing/*` |
| `modules/core/*` | `/modules/core/*` (one page per core module) |
| `modules/index.mdx` | `/modules` (catalog landing) |
| `modules/addons/{index.mdx,_category_.json}` | `/modules/addons` (landing) |
| `adrs/index.mdx` | `/adrs` (landing) |
| `api/index.mdx` | `/api` (landing) |

What does **not** live here:

| Path on the site | Source |
|---|---|
| `/modules/addons/<name>` | Synced from each addon repo's `README.md` (one repo per addon — see `orkestra-docs/sources.yaml`) |
| `/adrs/<NNNN>-*` | Synced from `docs/adr/*.md` in this repo, but only files with `public: true` in frontmatter |
| `/api/reference/**` | Generated by `docusaurus-plugin-openapi-docs` from `backend/openapi/enterprise.json` |

## Editing rules

- Standard Markdown / MDX with Docusaurus front matter (`title`, `sidebar_position`, `description`, …)
- Mermaid diagrams are supported (`@docusaurus/theme-mermaid` is wired in the site config)
- Custom React components in MDX work (`import Foo from '@theme/Foo'`) but cost portability — every reader on github.com sees raw JSX. Prefer plain Markdown when possible
- The site injects `custom_edit_url` at sync time so each page's "Edit this page" link points back to its source file in this repo — don't set `custom_edit_url` here manually

## How a change reaches the live site

```
monorepo/docs/site/*.mdx  ──merge to main──▶  orkestra-cc/orkestra (main)
                                                       │
                            nightly cron OR repository_dispatch from monorepo CI
                                                       │
                                                       ▼
                                        orkestra-cc/orkestra-docs build
                                        (runs npm run sync, then docusaurus build)
                                                       │
                                                       ▼
                                              docs.orkestra.cc
```

Lag on the auto path is up to 24 h (nightly cron). To deploy immediately after a doc change merges to monorepo `main`, run:

```bash
gh workflow run "Build & Deploy" -R orkestra-cc/orkestra-docs --ref main
```
